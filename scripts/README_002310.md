# 东方新能(002310)每日跟踪 - 运维手册

## 🎯 目标
确保每日跟踪任务稳定运行，即使单个数据源失败也能产出可用报告。

## 📋 系统组件

### 1. 健康检查脚本
**文件**: `scripts/health_check_002310.py`

**功能**: 任务执行前预检所有数据源
- Tavily Search API 可用性
- 财经网站连通性（东方财富、雪球、新浪、巨潮）
- 输出整体健康度（0-100%）

**执行示例**:
```bash
python3 scripts/health_check_002310.py
```

### 2. 增强版跟踪脚本
**文件**: `scripts/daily_002310_enhanced.py`

**功能**:
- 自动运行健康检查
- 主数据源：Tavily Search（多维度搜索）
- 备用数据源：agent-browser headed 模式（需配置）
- 优雅降级：即使所有数据源失败也输出框架报告
- 自动保存到本地 workspace

### 3. 数据源冗余矩阵

| 数据源类型 | 优先级 | 工具/API | 超时 | 重试 | 失败处理 |
|-----------|--------|----------|------|------|----------|
| 全网搜索 | 1 | Tavily Search | 15s | 3次 | 切换备用 |
| 股票数据 | 1 | agent-browser | 30s | 2次 | 使用缓存 |
| 公告/研报 | 2 | 巨潮资讯 | 20s | 2次 | 提供手动链接 |
| 资金流向 | 2 | agent-browser | 30s | 1次 | 标注缺失 |

## 🚨 故障检测与告警

### 自动检测机制
1. **单次失败检测**：每个工具调用记录 success/failure + 错误信息
2. **连续失败统计**：记录每个数据源的失败次数（需扩展 `/root/.openclaw/workspace/monitoring/`）
3. **健康度阈值**：<80% 触发警告，<50% 发送告警到群聊

### 告警规则（建议配置）
- **黄色告警**：单个数据源连续3次失败 → 群聊通知
- **红色告警**：健康度<50% → 私信通知 + 提供手动操作指南
- **恢复通知**：故障数据源恢复正常 → 群聊通报

## 🔄 优雅降级策略

### 级别1：部分数据成功
- 整合所有成功获取的片段信息
- 标注每个数据点的置信度（高/中/低）
- 仍输出完整报告结构

### 级别2：全部失败
- 输出框架报告，所有内容标注"❌ 数据暂缺/获取失败"
- 提供手动查询链接清单
- 给出明确操作建议

### 级别3：健康检查失败（避免雪崩）
- 跳过耗时的网站访问，直接进入降级模式
- 记录失败原因供后续分析
