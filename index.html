⚙️ System Prompt 结构 promptMode

  • • Tooling - 当前工具列表 + 简短描述
  • • Safety - 安全护栏提醒
  • • Skills - 可用技能元数据（按需加载）
  • • Workspace - 工作目录路径
  • • Documentation - 本地文档路径
  • • Sandbox - 沙箱环境信息（如启用）
  • • Current Date & Time - 时区与时间格式
  • • Runtime - 主机、OS、Node、模型、思考级别
  • • Reasoning - 推理可见性控制

三种 Prompt Mode

Mode: full (默认)
  - 包含所有章节
  - 主会话使用
  - 完整上下文：技能、内存、自更新、回复标签、心跳等

Mode: minimal (子代理)
  - 省略：Skills、Memory Recall、Self-Update、Model Aliases、User Identity、Reply Tags、Messaging、Silent Replies、Heartbeats
  - 保留：Tooling、Safety、Workspace、Sandbox、Date/Time、Runtime、注入的上下文

Mode: none
  - 仅返回基础身份行
  - 用于极简场景

🔧 工具定义与成本

  • • Tool List Text - 系统提示中的人类可读描述

  • ⚠️ Schema 计入上下文，即使不可见
  • • 每个工具都有独立的 schema 开销

🔄 自动 Compaction 机制

  • • 当会话接近或超出上下文窗口时自动触发
  • • 将较旧对话 总结 为紧凑条目
  • • 保留近期消息不变
  • • 摘要持久化到 JSONL 历史
  • • 可配置目标令牌数、压缩模型等

openclaw.json 配置示例

{
  "agents": {
    "defaults": {
      "compaction": {
        "mode": "auto",
        "targetTokens": 8000,
        "model": "openrouter/anthropic/claude-sonnet-4-5",
        "identifierPolicy": "strict"
      }
    }
  }
}

✂️ Session Pruning 会话剪枝

  • • 仅影响内存：每次 LLM 调用前临时修剪
  • • 不修改 磁盘上的 JSONL 历史
  • • 目标：减少 Anthropic 缓存失效
  • • 修剪对象：旧的 toolResult 消息
  • • 保护：用户/助手消息、含图片的结果
  • • 模式：cache-ttl（基于 TTL 自动触发）

Soft-trim vs Hard-clear

Soft-trim:
  - 对过大的 toolResult 仅保留头尾
  - 插入 "..." 并标注原始大小
  - 跳过含图片的结果

Hard-clear:
  - 完全替换为 "[Old tool result content cleared]"
  - 当工具结果超过 hardClearRatio 阈值时触发

默认配置:
  ttl: "5m"
  keepLastAssistants: 3
  softTrimRatio: 0.3
  hardClearRatio: 0.5

Plugin Hook 注册

module.exports = {
  hooks: {
    beforePromptBuild: async (ctx) => {
      // 在系统提示前注入自定义文本
      ctx.prependSystemContext = {
        text: "当前环境: production v2.1"
      };
    },

    beforeCompaction: async (ctx) => {
      // 记录压缩事件
      console.log(`Compacting ${ctx.sessionId}`);
    }
  }
};

🌊 Streaming 流式输出

  • • Block streaming - 分块发送部分回复
  • • 配置：blockStreamingDefault、blockStreamingChunk
  • • 触发点：text_end 或 message_end
  • • 支持推理流（reasoning）单独输出
  • • Human delay - 模拟人类输入节奏

openclaw.json 流式设置

{
  "agents": {
    "defaults": {
      "blockStreamingDefault": true,
      "blockStreamingBreak": "text_end",
      "blockStreamingChunk": {
        "minChars": 50,
        "maxChars": 500
      },
      "humanDelay": 800
    }
  }
}

📥 消息队列 Queueing

  • • interrupt - 中断当前运行
  • • steer - 引导到当前运行
  • • followup - 排队等待下一轮
  • • collect - 收集批次后执行
  • • 会话串行化：同一会话同时仅一个运行
  • • 全局排队：可选的全局串行化

按渠道配置队列模式

{
  "messages": {
    "queue": {
      "default": "followup",
      "byChannel": {
        "whatsapp": "interrupt",
        "telegram": "steer",
        "discord": "collect"
      }
    }
  }
}

🔗 Session 会话管理

  • • 直接聊天 → main 会话
  • • 群组/频道 → 独立会话键
  • • 会话持久化在网关的 JSONL 文件中
  • • 多设备可映射到同一会话，但历史不完全同步
  • • Control UI/TUI 显示网关端的真实转录

会话控制命令

/new       - 创建新会话（新 session id）
/reset     - 清空当前会话历史
/status    - 查看会话状态、上下文使用、成本
/context   - 上下文详情（注入文件、工具开销）
/compact   - 手动触发压缩总结

完整数据流

Inbound Message

用户消息通过 WhatsApp/Telegram/Discord 等渠道到达网关 → 路由到对应 sessionKey

Context Assembly

1. 加载 bootstrap 文件
2. 构建 system prompt (tooling + skills + runtime)
3. 合并会话历史 + 工具结果
4. 应用 compaction/pruning

LLM Call

发送完整提示到模型 → 流式返回助手 deltas → 检测工具调用

Tool Execution

执行工具（read/exec/browser 等） → 结果注入会话历史 → 继续流式输出

Response & Persistence

最终回复发送到用户渠道 → 完整运行元数据保存 → 上下文供下次使用

💡 关键要点

  • • OpenClaw 拥有自己的 系统提示构建系统，不使用 pi-agent 默认提示
  • • 引导文件 自动注入 到系统提示，消耗上下文窗口
  • • 工具列表（文本）+ 工具 schemas（JSON） 双重开销
  • • Skills 仅注入元数据，按需读取 SKILL.md
  • • Compaction 持久化压缩历史；Pruning 临时修剪内存（仅针对 Anthropic 缓存）
  • • 使用 /context、/status、/compact 管理上下文窗口
  • • hook 系统允许在 提示构建前 或 工具调用前后 拦截修改
