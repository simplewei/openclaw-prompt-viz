# child-companion 消息处理规范

## 消息格式

家长发送消息时，使用以下格式（任一）：

### 格式A：带前缀的指令（推荐）
```
@child-companion [命令] [参数]
```

### 格式B：关键词触发（备用）
```
记录 [星期] 聊天 [日期]
[内容]
```

---

## 支持的命令

### 1. 日常聊天记录
```
@child-companion 记录
【[星期] 聊天 [日期]】
主题：____
心情：__/10
开心事：____
困难：____
新学：____
观察：__
```

**示例**：
```
@child-companion 记录
【周五 家庭 2026-03-27】
心情：9/10（因为妈妈做了好吃的）
想家时刻：午睡时想妈妈
感恩：1.阳光 2.午饭 3.晚上聊天
周末计划：周六去公园
观察：亲子连接良好，孩子愿意分享
```

### 2. 生成周报
```
@child-companion 生成第1周周报
```
系统会自动：
- 扫描本周7天聊天记录
- 分析情绪/学业/社交/健康4维度
- 输出Markdown到 reports/weekly_周报_YYYY-MM-DD.md
- 可选发布到钉钉文档

### 3. 查询记录
```
@child-companion 查看 本周记录
@child-companion 查看 语录
@child-companion 查找 关键词
```

### 4. 初始化档案
```
@child-companion 设置档案
- 姓名：__
- 年龄：__
- 年级：__
- 身高体重：__
- 兴趣班：__
```

## 自动化流程

### 每日处理
1. 家长发送记录 → 我提取要点，追加到 memories/ 对应文件
2. 更新 session-state.md 的"最近活动"
3. 如果当天是周五，标记"周末计划待跟进"

### 每周日（20:00）
1. 检查本周聊天记录数量
2. 如果≥5天，自动生成周报
3. 将周报保存并发送钉钉文档
4. 更新 MEMORY.md 的Deliverables部分
5. 重置本周计数器

### 每月末
类似周报，但生成月报和季度评估

---

## 文件存储规则

### 聊天记录存储
- 每条记录追加到：`memories/[类别]_[描述].md`
- 按日期分类，使用 `### YYYY-MM-DD` 标题
- 记录格式：关键词为主，避免长文

### 周报存储路径
```
reports/weekly_周报_[周日日期].md
reports/weekly_周报_[周日日期]_已发布.md（发布后重命名）
```

### 文档发布
- 使用 mcporter 调用钉钉文档API
- 目标文件夹：openclaw（parentDentryUuid: ndMj49yWjXNaR09OIDDmZwy3J3pmz5aA）
- 命名：`[领域]_[内容]_[YYYY-MM-DD]`
- 返回文档链接给家长

## 状态管理

### session-state.md 更新时机
- 每次记录处理后（更新"最近活动"）
- 周报生成后（更新"近期报告"表格）
- 每月1日（重置月度统计）

### heartbeats.md 集成
我已添加"孩子陪伴师周报生成"任务，每周日20:00自动触发。

## 错误处理

### 缺少记录
如果周日检查发现本周记录<5天：
- 发送提醒："本周聊天记录不足5天，请补录后重试‘生成周报’"
- 不生成不完整周报

### 子代理不可用
回退到main会话直接处理，功能不变
